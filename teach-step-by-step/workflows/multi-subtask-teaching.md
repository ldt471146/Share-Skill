# Multi-Subtask Teaching Workflow

用于多资料、多产物，或 3 个以上相对独立教学子任务。目标是保持教学连续性，同时避免把多个独立任务混在同一个上下文里。

## Trigger

满足任一条件时使用：

- 用户同时要求教学多个文件、多个概念或多个产物。
- 任务包含 3 个以上相对独立子任务。
- 当前任务会显著挤占上下文，影响后续逐步教学。
- 任务同时包含资料拆解、实现、验证、复盘等多个阶段。

## Steps

1. 先列出子任务清单，每个子任务只写一句话。
2. 判断子任务之间是否相互依赖。
3. 先执行最能支撑用户理解的一个子任务，不要一次展开全部。
4. 每完成一个子任务，回到主线总结它和整体目标的关系。
5. 如果环境支持独立 agent 或新会话，可把互不依赖的资料扫描、验证、对比工作隔离出去；否则在当前上下文中按清单顺序模拟。
6. 始终遵守 `rules/teaching-principles.md`，不要为了并行而牺牲逐步教学。

## Done Check

- 子任务清单明确。
- 当前只推进了一个最小教学单元。
- 用户能看出当前子任务和总目标的关系。
- 没有把多个独立解释混成一段长讲解。

## Avoid

- 不要把教学拆得支离破碎，导致用户失去主线。
- 不要在用户没有确认前并行讲多个概念。
- 不要为了使用 multi-subtask 而制造子任务。
